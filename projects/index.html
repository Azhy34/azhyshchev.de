Web Sales Assistant — da-vinchi.pl
RAG · AI Agent · E-commerce · Production

Production AI Sales Assistant for a Polish wallpaper and décor e-commerce store. A LangGraph ReAct agent with 5 tools, Hybrid RAG architecture (SQL for hard facts + pgvector for semantic knowledge), ETL pipelines, buyer-scenario tests, and a web chat widget interface (FastAPI /chat endpoint). Production-grade, not a demo.

1,200+ SKU Zero hallucinated prices. Hard facts stored in SQL, not vector search.
~90% API cost reduction vs. hot-cache approach through architectural design.
5 tools LangGraph ReAct agent: search, filter, detail, recommend, FAQ.
98% Data optimisation through ETL pipeline and knowledge-base contract tests.

Architecture

  • Hybrid RAG: SQL for deterministic facts (price, SKU, dimensions), pgvector for semantic knowledge (style, mood, recommendations).
  • LangGraph ReAct agent — tool-calling loop with fallback handling and observability.
  • FastAPI backend + web chat widget (/chat endpoint). Telegram used for internal debugging only.
  • ETL pipeline with deduplication and knowledge-base contract tests.

Business impact

  • Handles product search, filtering, detail queries, recommendations, and FAQ without human intervention.
  • Buyer-scenario integration tests ensure the agent behaves correctly under edge cases.
  • Production-deployed on Railway — not a prototype.

Tech stack

Python LangGraph FastAPI Supabase pgvector Gemini Sentence-Transformers LangSmith Docker Railway pytest

Invoice Automation Pipeline
Automation · LLM Pipeline · Production

Invoices from multiple suppliers arrive as PDFs by email in different formats. The manual routine was: find email, download PDF, classify supplier, extract fields, file to Drive, enter into accounting system. Now it runs on a schedule — automatically, end-to-end.

2-layer Baseline parser first, Gemini verification second. Works even when API is down.
Cron-ready GitHub Actions scheduling, deduplication, status tracking, Telegram summaries.
Zero reruns Two-level deduplication prevents duplicate records from forwarded attachments.
Multi-supplier Different PDF formats per supplier — handled by the LLM extraction layer.

Pipeline flow

  • IMAP inbox scan → PDF attachment detection → supplier classification.
  • Baseline regex parser extracts structured fields first.
  • Gemini 2.5 Flash verifies and fills gaps, handles format variations.
  • Validated records upserted to Supabase, filed to Google Drive.
  • Telegram summary with run results and any flagged anomalies.

Engineering decisions

  • Reliability over extraction accuracy — the pipeline never silently fails.
  • Deterministic baseline ensures LLM is a verification layer, not a dependency.
  • Production-grade: idempotent, logged, scheduled, and monitored.

Tech stack

TypeScript Node.js Gemini 2.5 Flash Supabase Google Drive API IMAP GitHub Actions Telegram Bot API

AI Image Composer
Full-Stack · Multimodal AI · Production

A Polish creative studio was spending hours on trial-and-error prompting to produce personalized AI paintings for clients. The bottleneck was identity replacement — getting the client's face into a scene without quality loss. Built a production web app in one day that turns this into a 30-second workflow.

30 sec Per artwork from upload to result. Was hours of manual work per image.
1 day From requirement to production deployment. Built with React 19 + Gemini.
85-90% Daily team adoption — because the tool was simpler than the manual process.
Mobile-first Optimized for phones and field usage, not just desktop demo behavior.

How it works

  • User uploads source photo + selects style template.
  • Canvas API extracts face region with masking.
  • Gemini multimodal API performs identity-preserving style transfer.
  • Result delivered in the browser within seconds.

Engineering choices

  • Chose AI generation over geometric transformation — adoption mattered more than technical elegance.
  • React 19 for instant UI feedback without page reloads.
  • Railway deployment for zero-config backend scaling.

Tech stack

React 19 TypeScript Gemini Canvas API Node.js Railway

B2B Lead Pipeline
Automation · B2B Ops · Internal tool

Germany-focused outreach pipeline that turns a Google Maps search query into CRM-ready lead records. Automated company discovery, Impressum extraction, contact enrichment, and deduplication. Replaced a multi-hour manual research process with a repeatable 15-25 minute workflow.

15-25 min Typical pipeline run time for 80-120 companies from search to CRM-ready output.
~250 lines Lean Python implementation with strong practical output and Supabase deduplication.
Multi-layer Apify scraper + Tavily / Exa search + Gemini extraction + Pandas cleaning.
Germany-specific Impressum parsing for contact extraction — compliant with German business structure.

Pipeline flow

  • Google Maps search via Apify → company list with basic metadata.
  • Tavily + Exa search for each company website → Impressum page discovery.
  • Gemini extracts structured contact fields from unstructured Impressum text.
  • Pandas cleaning + deduplication → Supabase upsert → CSV export.

Use case

  • Built for B2B outreach in German market — kitchen studios, home appliance retailers, interior design firms.
  • Output is ready for mail merge or CRM import without manual cleanup.
  • Combined multiple search and extraction layers to improve coverage and data quality.

Tech stack

Python Apify Tavily Exa Gemini Supabase Pandas

Virtual Wallpaper Try-On
Full-Stack · Multimodal AI · Business Process Automation · Production

A wallpaper manufacturer's sales team was spending 4-6 hours daily creating client visualizations in Photoshop — manual perspective correction, lighting adjustments, pattern alignment for every single order. Built a production web app that replaces this with a 2-minute AI-generation workflow.

75% Manual work reduction. 4-6 hours/day → 15-20 minutes/day.
<2 min Per mockup. Was 2-4 hours of Photoshop work per visualization.
85-90% Daily team adoption — because the tool was simpler than manual work.
Same day Client turnaround. Was 2-3 days for a single mockup set.

Engineering decisions

  • Prototyped two approaches: geometric transformation (SAM2 segmentation) vs. AI generation. AI won — not because it's more sophisticated, but because non-technical sales team members actually use it.
  • Gemini multimodal handles perspective, lighting, and pattern placement automatically — no masks or corrections needed.
  • Mobile-first design: sales team uses phones on client visits.
  • Multi-office billing: client-side API key injection for separate Poland / Ukraine office quotas.

Business impact

  • Unlimited parallel mockup processing — no longer limited by designer availability.
  • Same-day client delivery replaced the 2-3 day turnaround.
  • 85-90% daily adoption from day one — adoption over elegance.
  • Sales team can now handle more clients with the same headcount.

Tech stack

React 19 Node.js Gemini (image-preview) Firebase Canvas API Vercel GitHub CI/CD
